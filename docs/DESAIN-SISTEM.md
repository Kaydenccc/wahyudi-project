## 8. Keamanan Sistem

### 8.1 Autentikasi & Otorisasi

| Mekanisme | Implementasi |
|-----------|-------------|
| Password Hashing | bcryptjs dengan 12 salt rounds |
| Session Token | JWT (JSON Web Token) via jose library |
| Cookie | httpOnly, secure, sameSite=lax, 7 hari |
| Role-based Access | 4 role dengan hak akses berbeda |
| Middleware | Cek token & role di setiap request API |

### 8.2 Validasi Input

| Mekanisme | Implementasi |
|-----------|-------------|
| Schema Validation | Zod schema di setiap API endpoint |
| ObjectId Validation | mongoose.isValidObjectId() |
| Regex Escaping | Escape karakter khusus pada search query |
| Field Whitelisting | Hanya field yang diizinkan bisa diupdate |
| File Upload | Validasi tipe (PNG/JPG) dan ukuran (maks 5MB) |

### 8.3 Security Headers (via next.config.ts)

| Header | Nilai | Fungsi |
|--------|-------|--------|
| X-Frame-Options | DENY | Cegah clickjacking |
| X-Content-Type-Options | nosniff | Cegah MIME sniffing |
| Referrer-Policy | strict-origin-when-cross-origin | Kontrol referrer |
| X-XSS-Protection | 1; mode=block | Cegah XSS di browser lama |
| Permissions-Policy | camera=(), microphone=() | Batasi akses hardware |

### 8.4 Infrastruktur

| Mekanisme | Implementasi |
|-----------|-------------|
| HTTPS | SSL/TLS via Let's Encrypt + Nginx |
| Firewall | UFW (SSH, HTTP, HTTPS only) |
| SSH Key Auth | Login VPS tanpa password |
| Standalone Output | Minimized attack surface |
